    CLI Reference

    Complete reference for all vpnconnect commands, subcommands, and flags.

    Tip: Run vpnconnect <command> --help at any time to see inline help for that command.

    Global flags

    These flags are available on every command.

    Flag Default Description
    --log-level warn Log verbosity. One of: trace, debug, info, warn, error

    vpnconnect

    vpnconnect [command] [flags]

    VPN client with TUN/TAP interface management and Pulse protocol support. If no command is given, the graphical UI is launched (equivalent to vpnconnect ui).

    connect

    Establish a VPN connection to a Juniper Pulse endpoint using a saved profile.

    vpnconnect connect [flags]

    Flags

    Flag Default Description
    --profile "" Name of the profile to use. If omitted, the default profile is used
    --standalone false Run without the daemon. The connection is managed entirely in the current process. Requires root/administrator privileges
    --prompt-password false Prompt for a password even when one is stored in the OS keychain
    --merge-dns false Merge the profile's DNS list with the server-provided DNS list instead of replacing it
    --mfa-automate false Automate Microsoft SSO credential entry during MFA. Overrides the profile's mfa_automate setting for this invocation

    Notes

    • In normal (non-standalone) mode connect sends the connection request to the running daemon and returns immediately. The daemon manages the interface lifecycle.
    • In --standalone mode the process blocks until interrupted (Ctrl+C / SIGTERM). See How to connect in standalone mode.
    • If the profile has MFAEnabled: true and no cached DSID cookie is available, the MFA browser flow is triggered automatically before sending the connection request. By default the browser waits for manual login; pass --mfa-automate to have credentials filled automatically (Microsoft SSO only).

    Examples

    # Connect using the profile named "work"
vpnconnect connect --profile work

# Connect without a daemon, running in the foreground
sudo vpnconnect connect --profile work --standalone

# Always prompt for a password (ignores stored keychain entry)
vpnconnect connect --profile work --prompt-password

# Enable Microsoft SSO automation for this connection only
vpnconnect connect --profile work --mfa-automate

    disconnect

    Disconnect from an active Pulse VPN connection and restore network settings.

    vpnconnect disconnect [flags]

    Flags

    Flag Default Description
    --profile "" Profile name (optional). When omitted, the active connection is disconnected

    Examples

    vpnconnect disconnect
vpnconnect disconnect --profile work

    status

    Query the daemon for the current VPN connection state and details.

    vpnconnect status

    No flags. Prints the current connection state and per-connection details (endpoint, interface, IP address, etc.) reported by the daemon.

    Examples

    vpnconnect status

    statistics

    Query the daemon for live packet and byte counters for the active VPN connection.

    vpnconnect statistics

    No flags. Prints transmitted and received packet/byte counts.

    Examples

    vpnconnect statistics

    mfa

    Perform only the MFA (browser) authentication step for a profile and cache the resulting DSID cookie. Useful for pre-authenticating before a connect without triggering the browser popup at connect time.

    vpnconnect mfa --profile <name>

    Flags

    Flag Required Default Description
    --profile Yes — Profile name to authenticate
    --mfa-automate No false Automate Microsoft SSO credential entry. Overrides the profile's mfa_automate setting for this invocation

    Examples

    vpnconnect mfa --profile work

# Enable Microsoft SSO automation for this pre-authentication only
vpnconnect mfa --profile work --mfa-automate

    profile

    Manage VPN profiles. Profiles store the endpoint URL, username, and connection options. Passwords are stored separately in the OS keychain.

    vpnconnect profile <subcommand> [flags]

    Subcommands

    Subcommand Description
    add Create a new profile
    list List all saved profiles
    remove Delete a profile
    rename Rename a profile
    set-password Set or update the keychain password for a profile

    profile add

    Create a new Pulse VPN profile.

    vpnconnect profile add --name <name> --endpoint <url> --username <user> [flags]

    Flags

    Flag Required Default Description
    --name Yes — Unique profile name
    --endpoint Yes — VPN endpoint URL (e.g. https://vpn.example.com)
    --username Yes — Username for authentication
    --password No "" Password. If omitted and --store-password is true, you will be prompted
    --mfa No false Enable browser-based MFA for this profile
    --mfa-automate No false Automate Microsoft SSO credential entry during MFA. Requires --mfa. Tailored to the Microsoft Azure AD login page
    --store-password No true Store the password in the OS keychain
    --merge-dns No false Merge profile DNS with server DNS instead of replacing

    Examples

    # Basic profile, password stored in keychain
vpnconnect profile add --name work --endpoint https://vpn.example.com --username alice

# Profile with MFA (manual mode — works with any identity provider)
vpnconnect profile add --name work-mfa \
  --endpoint https://vpn.example.com \
  --username alice \
  --mfa \
  --store-password=false

# Profile with MFA and automated Microsoft SSO credential entry
vpnconnect profile add --name work-mfa \
  --endpoint https://vpn.example.com \
  --username alice@example.com \
  --mfa --mfa-automate \
  --store-password=false

    profile list

    Display all saved profiles.

    vpnconnect profile list

    No flags. Prints a table with name, endpoint, username, MFA status, merge-DNS setting, and creation date.

    profile remove

    Delete a profile and its associated credentials.

    vpnconnect profile remove --name <name>

    Flags

    Flag Required Description
    --name Yes Name of the profile to delete

    You will be asked to confirm before deletion.

    profile rename

    Rename a profile.

    vpnconnect profile rename --old-name <name> --new-name <name>

    Flags

    Flag Required Description
    --old-name Yes Current profile name
    --new-name Yes New profile name

    profile set-password

    Set or update the password stored in the OS keychain for a profile.

    vpnconnect profile set-password --name <name> [--password <password>]

    Flags

    Flag Required Description
    --name Yes Profile name
    --password No New password. If omitted, you will be prompted

    server

    Start the VPNConnect background daemon. The daemon listens on a random local TCP port and manages the VPN interface lifecycle on behalf of connect, disconnect, and other client commands.

    sudo vpnconnect server

    No flags. Requires root / administrator privileges. Intended to be run as a system service (systemd, launchd, Windows Service). See How to install VPNConnect for service configuration.

    The daemon writes a server info file (port + auth token) at a platform-specific path so that client commands can locate it:

    Platform Path
    Linux /run/vpnconnect/server.json
    macOS /var/run/vpnconnect/server.json
    Windows %ProgramData%\vpnconnect\server.json

    tuntap

    Manage TUN/TAP network interfaces directly, bypassing the daemon. Useful for debugging or scripted environments.

    vpnconnect tuntap <subcommand> [flags]

    Subcommands

    Subcommand Description
    create Create and configure a TUN/TAP interface
    remove Remove an existing interface
    list List all TUN/TAP interfaces on the system

    tuntap create

    Create and configure a new TUN/TAP interface. Requires root/administrator privileges.

    vpnconnect tuntap create --name <name> --type <tun|tap> --ip <addr> --netmask <mask> [flags]

    Flags

    Flag Required Default Description
    --name Yes — Interface name (e.g. vpn0, tun1)
    --type Yes — Interface type: tun or tap
    --ip Yes — IP address to assign
    --netmask Yes — Netmask (e.g. 255.255.255.0)
    --mtu No 0 MTU for the interface. 0 uses the system default
    --routes No — Additional routes to add (repeatable, CIDR notation)
    --dns No — DNS servers to configure (repeatable)
    --up-script No "" Script to run after the interface is up
    --down-script No "" Script to run before the interface is removed
    --persist No false Keep the interface after the process exits
    --daemon No false Block and keep the interface up until interrupted

    Examples

    sudo vpnconnect tuntap create \
  --name vpn0 \
  --type tun \
  --ip 10.0.0.2 \
  --netmask 255.255.255.0 \
  --mtu 1400 \
  --daemon

    tuntap remove

    Remove an existing TUN/TAP interface.

    vpnconnect tuntap remove --name <name>

    Flags

    Flag Required Description
    --name Yes Interface name to remove

    tuntap list

    List all TUN/TAP interfaces currently present on the system.

    vpnconnect tuntap list

    No flags.

    ui

    Launch the graphical user interface. This is the default command when vpnconnect is invoked with no arguments.

    vpnconnect ui

    No flags. Opens the Wails-based desktop application. Only one instance is permitted (enforced via UniqueID: com.vpnconnect.ui); a second invocation will focus the existing window.

    version

    Print build version information.

    vpnconnect version

    No flags. Prints the version string, git commit hash, and build date.

    Example output

    vpnconnect version: 1.0.0
commit: abc1234
build date: 2026-04-10
