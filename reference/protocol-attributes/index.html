    Pulse Protocol Attributes

    This reference describes the Pulse (Juniper) VPN configuration attributes sent by the server during the configuration phase of a connection. It lists which attributes are supported, ignored, or unknown in vpnconnect.

    0x0001 Internal IPv4 Address Supported Legacy IPv4 assignment
    0x0002 Internal IPv4 Netmask Supported Netmask for internal IP
    0x0003 IPv4 DNS Server Supported Primary / secondary DNS

    Group notes

    Network layer configuration (0x0001–0x0010)

    Define the core IP configuration for the VPN tunnel. vpnconnect supports both IPv4 and IPv6 assignment, DNS servers, and split-tunnelling routes.

    ESP security parameters (0x4010–0x4017)

    Define the parallel ESP (UDP) tunnel used for higher throughput. They specify encryption and integrity algorithms, rekeying intervals, and anti-replay protection. vpnconnect uses these to negotiate the ESP session; see Pulse protocol internals for details.

    Pulse ESP "Unstupid" Mode (0x4024)

    Normally Pulse requires IPv6 traffic over IPv6 transport and IPv4 over IPv4. When set to 1 this attribute indicates the server supports cross-protocol tunnelling (e.g. sending IPv6 packets over an IPv4 ESP tunnel). If 0 and a cross-protocol packet must be sent, vpnconnect falls back to the IF-T/TLS data channel.

    Routing and proxy policies (0x4000, 0x4001, 0x401f, 0x4022)

    Communicate the server's desired client-side policies. 0x4000/0x4001 prevent dual-homing or ensure the tunnel route is preferred. 0x4022 and related attributes control whether the client should use, ignore, or preserve system-wide proxy settings. These are currently ignored.

    Informational markers (0x4025, 0x4026)

    Sent by newer Pulse Secure / Ivanti Connect Secure servers to advertise version capabilities (e.g. 9.1R14+). They carry no configuration payload but are useful for debugging protocol version compatibility.
