    How to build VPNConnect from source

    This guide shows how to compile VPNConnect on Linux, macOS, and Windows. Building from source is intended for developers and contributors.

    Prerequisites

    All platforms

    • Go 1.25.5+ — Download
    • Node.js 18+ and npm — required to build the Wails frontend
    • Task (go-task) — Install
    • goreleaser v2 — Install
    • CGO enabled — the build requires C compiler toolchain

    Linux

    # Debian/Ubuntu
sudo apt install build-essential libgtk-3-dev libwebkit2gtk-4.1-dev

    macOS

    • Xcode Command Line Tools: xcode-select --install
    • Minimum deployment target: macOS 14.0
    • Required link flags: -framework UniformTypeIdentifiers

    Windows

    • MSYS2 / MinGW-w64 (for CGO)
    • The Wintun driver DLL is downloaded automatically by the build; no manual setup needed.

    Clone the repository

    git clone https://github.com/VPNConnectDev/vpnconnect.git
cd vpnconnect

    Build for the current platform

    task build-single-target

    This runs goreleaser build --snapshot --clean --single-target, which:

    1. Runs go mod tidy.
    2. Installs frontend npm dependencies (npm install in cmd/vpnconnect/ui/frontend).
    3. Builds the Vite frontend (npm run build).
    4. Compiles the Go binary with the desktop and production build tags.

    The resulting binary is placed in the dist/ directory.

    Build a release package

    task release-single-target

    Runs goreleaser release --snapshot --clean --single-target. In addition to the binary, this produces the platform-specific installer package (.deb, .rpm, .pkg, .exe installer) in dist/.

    Direct Go build (advanced)

    If you want to avoid goreleaser and build directly:

    # Build the frontend first
cd cmd/vpnconnect/ui/frontend
npm install
npm run build
cd ../../../..

# Build the binary
go build -tags=desktop,production -o bin/vpnconnect .

    Linux-specific flags

    No additional flags required beyond CGO.

    macOS-specific flags

    CGO_CFLAGS="-mmacosx-version-min=14.0" \
CGO_LDFLAGS="-mmacosx-version-min=14.0 -framework UniformTypeIdentifiers" \
go build -tags=desktop,production -o bin/vpnconnect .

    Windows-specific flags (GUI build)

    go build -tags=desktop,production -ldflags="-H windowsgui" -o bin/vpnconnect.exe .

    Cross-compilation

    Cross-compilation requires Docker and the cross-compilation image:

    task setup:docker

    This builds the Docker image (~800 MB download) with the required toolchains for Linux, macOS, and Windows. After setup:

    docker run --rm -v "$PWD":/src vpnconnect-cross goreleaser build --snapshot --clean

    Running tests

    go test ./internal/...

    Some tests require root privileges (TUN interface creation). Platform-specific test files use build tags (e.g. example_darwin_test.go).
