    Getting started: your first VPN connection

    In this tutorial we will install VPNConnect, create a VPN profile, connect to the VPN, verify the connection is working, and then disconnect cleanly. By the end you will have made a complete round-trip and will know the core workflow.

    We will use the CLI throughout. The UI app wraps the same operations in a graphical interface; once you understand the CLI, the UI will feel familiar.

    What you will need

    • A computer running Linux, macOS, or Windows.
    • Administrator / root access to install the daemon.
    • The address of a Pulse Secure VPN endpoint, a username, and a password (provided by your organisation's IT department).

    Step 1 — Install VPNConnect

    Download and install the package for your operating system from the Releases page.

    Linux (Debian/Ubuntu):

    sudo dpkg -i ~/Downloads/vpnconnect_<version>_amd64.deb

    macOS:

    Open the .pkg installer and follow the prompts.

    Windows:

    Run the .exe installer.

    After installation the VPNConnect daemon starts automatically as a system service. You can confirm it is running:

    # Linux
systemctl status vpnconnect-server

# macOS
sudo launchctl list | grep vpnconnect

# Windows (elevated prompt)
sc query VPNConnect

    You should see the service in a running state. If it is not running, start it:

    # Linux
sudo systemctl start vpnconnect-server

# macOS
sudo launchctl load /Library/LaunchDaemons/org.vpnconnect.server.plist

    Step 2 — Create a VPN profile

    A profile stores the endpoint address, your username, and connection options. Run:

    vpnconnect profile add \
  --name my-first-vpn \
  --endpoint https://vpn.example.com \
  --username alice

    Replace https://vpn.example.com with the address of your VPN gateway and alice with your username.

    You will be prompted for a password:

    Enter password (leave empty to skip):
✓ Password stored securely in keychain
✓ Profile 'my-first-vpn' created successfully

    Verify the profile was saved:

    vpnconnect profile list

    You should see your new profile in the output table.

    Step 3 — Connect

    vpnconnect connect --profile my-first-vpn

    vpnconnect loads the profile, retrieves the password from the keychain, and sends the connection request to the daemon. The daemon authenticates with the VPN server and brings up the TUN interface.

    On success you will see:

    ✓ Connected via server

    If your organisation uses MFA (Microsoft SSO): Add --mfa to the profile add command and see How to connect with MFA. A browser window will open the first time you connect; complete the MFA prompt and it will close automatically.

    Step 4 — Verify the connection

    Check the connection status:

    vpnconnect status

    You should see output like:

    Status: ok
Connection details:
  state: connected
  profile: my-first-vpn
  interface: tun0
  ip: 10.100.5.42
  endpoint: https://vpn.example.com

    Notice the ip field — that is the address assigned to you on the VPN network. You can also check whether VPN-only resources are reachable (e.g. ping an internal server your IT team has told you about).

    Step 5 — View live statistics

    While connected, observe how many packets have been exchanged:

    vpnconnect statistics

    Statistics status: ok
Packet statistics:
  {
    "rx_bytes": 12840,
    "rx_packets": 95,
    "tx_bytes": 9320,
    "tx_packets": 88
  }

    Run it again in a few seconds. You should see the counters increase as background traffic flows through the tunnel.

    Step 6 — Disconnect

    vpnconnect disconnect

    ✓ Disconnected from Pulse VPN (managed by server)

    VPNConnect sends a proper logout packet to the server (releasing your session slot immediately), removes the VPN routes, and tears down the TUN interface. Your network returns to its normal state.

    What you have learned

    • VPNConnect is split into a privileged daemon and unprivileged clients (CLI, UI).
    • Profiles store connection settings; passwords live in the OS keychain.
    • connect, status, statistics, and disconnect are the core daily commands.
    • The daemon manages the interface lifecycle; clients just send commands.
